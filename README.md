# Gobernabilidad de TI - Grupo AJE

## Descripción
Este repositorio presenta una propuesta de gobernabilidad de TI para el Grupo AJE utilizando GitHub como plataforma de colaboración y control de versiones.

## Objetivo
Implementar buenas prácticas para la gestión del código, el trabajo colaborativo y el control de cambios mediante GitHub.

## Estructura del trabajo
- `README.md`: Información general del proyecto.
- `docs/`: Documentación de políticas y procesos.
- `app/`: Código fuente del proyecto.

## Flujo de trabajo
- **main:** Versión estable del proyecto.
- **develop:** Integración de nuevas funcionalidades.
- **feature/***: Desarrollo de nuevas características.

Los cambios se realizan en ramas `feature`, luego se envían mediante un **Pull Request** para su revisión antes de integrarse a `develop` y posteriormente a `main`.

## Roles
- **Administrador:** Gestiona el repositorio y aprueba cambios.
- **Desarrolladores:** Implementan nuevas funcionalidades y corrigen errores.
- **Lectores:** Pueden visualizar el repositorio sin realizar modificaciones.

## Herramientas utilizadas
- Git
- GitHub
- GitHub Proyects
- GitHub Issues
- GitHub Pull Requests
